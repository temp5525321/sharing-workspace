# Glip 서비스 기획 분석 리포트

> 분석일: 2026-04-05
> 대상: Glip (https://app.alpha.glip.studio)
> AI 생성 영상/이미지 공유 커뮤니티

---

## 1. 시리즈/에피소드/클립/라운지 컨셉 전달력 분석

### 1-1. 시리즈 (Series)

**컨셉**: 하나의 세계관이나 주제 아래 묶인 콘텐츠 묶음 (넷플릭스의 "시리즈" 개념)

| 항목 | 평가 |
|------|------|
| 직관성 | 중간. "시리즈"라는 단어 자체는 익숙하지만, AI 생성 영상 맥락에서는 "프로젝트"나 "작품"에 가까울 수 있음 |
| 유저 행동 예측 | 시리즈를 "만드는" 행위의 허들이 높음. 유저가 단일 영상만 올리고 싶은 경우 시리즈 생성이 강제되면 이탈 가능성 |
| 개선 포인트 | 시리즈 없이도 에피소드/클립 단독 업로드 가능하게 하되, 나중에 묶을 수 있는 구조 권장 |

**핵심 리스크**: AI 영상 크리에이터 대부분은 아직 "연작" 단위로 제작하지 않음. 시리즈 구조가 콘텐츠 업로드 허들을 높일 수 있음.

### 1-2. 에피소드 (Episode)

**컨셉**: 시리즈 내 개별 회차 (1화, 2화...)

| 항목 | 평가 |
|------|------|
| 직관성 | 높음. TV 시리즈의 에피소드 개념과 동일하여 이해도 높음 |
| 유저 행동 예측 | 시리즈가 활성화되어야 에피소드도 의미가 생김. 시리즈 없는 에피소드는 그냥 "영상"과 구분이 안 됨 |
| 개선 포인트 | 에피소드 = 독립 영상으로도 기능하되, 시리즈에 소속되면 순서가 생기는 유연한 구조 필요 |

**핵심 리스크**: 에피소드와 클립의 차이가 유저에게 명확하지 않을 수 있음. "긴 영상 = 에피소드, 짧은 영상 = 클립"이라는 기준이 있다면 업로드 시 가이드 필요.

### 1-3. 클립 (Clip)

**컨셉**: 짧은 하이라이트 또는 독립 숏폼 콘텐츠

| 항목 | 평가 |
|------|------|
| 직관성 | 높음. 틱톡/릴스/숏츠에 익숙한 유저에게 가장 자연스러운 포맷 |
| 유저 행동 예측 | 가장 낮은 진입장벽. AI 영상의 현실적 길이(5~60초)와도 잘 맞음 |
| 개선 포인트 | 클립이 메인 콘텐츠 단위가 되어야 초기 활성화에 유리. 홈 피드에서 클립 비중을 높일 것 |

**핵심 리스크 없음**: 오히려 클립이 Glip의 킬러 포맷이 될 가능성 높음. 현재 AI 생성 영상의 평균 길이(Sora, Kling, Runway 등)가 5~15초이므로 클립 중심 설계가 현실적.

### 1-4. 라운지 (Lounge)

**컨셉**: 외부 AI 영상(유튜브, 틱톡 등) 임베드 링크 공유 공간. 직접 영상을 제작하기 어렵지만 커뮤니티에 기여하고 싶은 유저를 위한 섹션이며, 서비스 초기에 이미 검증된 품질의 외부 영상을 대량으로 확보하기 위한 전략적 공간.

| 항목 | 평가 |
|------|------|
| 직관성 | 중간. "라운지"라는 네이밍만으로는 외부 영상 공유 공간이라는 점이 즉시 전달되지 않을 수 있으나, 진입 후 콘텐츠를 보면 이해 가능 |
| 유저 행동 예측 | 직접 제작 역량 없는 유저의 진입장벽을 크게 낮춤. "이거 봤어? 대박이야!" 식의 가벼운 공유 문화가 형성될 수 있음. 초기 콘텐츠 물량 확보에 매우 유리 |
| 개선 포인트 | 라운지가 "외부 AI 영상 큐레이션 공간"임을 명시하는 서브 설명 추가. "당신이 발견한 AI 영상을 공유해주세요" 같은 가이드 |

**전략적 가치**: 라운지는 단순 커뮤니티가 아니라 Glip의 초기 성장 엔진. (1) 외부 크리에이터의 콘텐츠로 플랫폼 품질을 보증하고, (2) 공유만으로 기여할 수 있어 일반 유저 유입을 촉진하며, (3) 향후 외부 크리에이터를 Glip 내부 크리에이터로 전환하는 퍼널 역할.

### 컨셉 전달력 종합

```
클립 (높음) > 에피소드 (중간) > 시리즈 (중간-낮음) > 라운지 (낮음)
```

**권장 방향**: 초기에는 클립 중심으로 유저 유입을 확보하고, 시리즈/에피소드는 파워 유저의 자연스러운 확장 경로로 포지셔닝.

## 2. 국내 시장 타겟 시 우선 강화 기능 및 바이럴 전략

### 2-1. 뾰족한 타겟 시장 정의

**1순위 타겟: AI 영상 크리에이터 (얼리어답터)**
- Sora, Kling, Runway, Pika 등 사용자
- 현재 결과물 공유 채널: X(트위터), 디스코드, 유튜브
- 페인포인트: 프롬프트/세팅 공유가 체계적이지 않음, 자신의 작품을 포트폴리오로 보여줄 곳 부재

**2순위 타겟: AI 영상 소비자 (구경꾼)**
- AI 영상에 관심 있지만 직접 만들지는 않는 층
- "와 이게 AI로 만든 거야?" 반응을 이끌어내는 콘텐츠 소비자

### 2-2. 바이럴을 위한 우선 강화 기능 (Top 5)

#### (1) 프롬프트 카드 시스템 (최우선)
- 영상과 함께 사용된 프롬프트, 모델명, 파라미터를 구조화된 카드로 표시
- "이 영상은 어떻게 만들었어?"에 대한 즉각적 답변 = 공유 동기
- 카드 자체가 SNS 공유 시 썸네일이 되도록 OG 이미지 자동 생성
- **바이럴 메커니즘**: X(트위터)에 공유 시 프롬프트 카드가 보이면 클릭 유도 극대화

#### (2) Before/After (원본 프롬프트 vs 결과물) 비교 뷰
- 같은 프롬프트로 다른 모델 결과 비교
- 프롬프트 반복 실험 과정 공유
- **바이럴 메커니즘**: "Sora vs Kling 같은 프롬프트 결과 비교" 콘텐츠는 SNS에서 높은 engagement

#### (3) 리믹스/리메이크 기능
- 다른 유저의 프롬프트를 가져와서 나만의 버전 만들기
- 원본 크리에이터에게 크레딧 자동 부여
- **바이럴 메커니즘**: 리믹스 체인이 길어질수록 원본의 노출도 증가 (TikTok 듀엣 효과)

#### (4) 주간 챌린지/테마전
- 매주 특정 테마 제시 (예: "빗속의 도시", "90년대 광고")
- 참여작 투표 → 위클리 랭킹
- **바이럴 메커니즘**: 챌린지 참여 자체가 콘텐츠 생산 동기 + SNS 공유 동기

#### (5) 임베드/공유 최적화
- 클립을 외부 사이트에 임베드할 수 있는 코드 제공
- X, 카카오톡, 블로그 공유 시 자동 재생 프리뷰
- **바이럴 메커니즘**: 콘텐츠가 플랫폼 밖에서도 유통되어야 유입 발생

### 2-3. 국내 시장 특화 전략

| 전략 | 구체적 실행 |
|------|------------|
| 카카오톡 공유 최적화 | OG 태그 한국어 최적화, 카톡 미리보기에서 영상 자동 재생 |
| 네이버 블로그/카페 연동 | 임베드 코드가 네이버 에디터에서 동작하도록 지원 |
| AI 영상 크리에이터 커뮤니티 공략 | 디스코드 한국 AI 영상 커뮤니티(예: AI 영상 연구소 등)와 파트너십 |
| 크리에이터 초대 프로그램 | X에서 AI 영상으로 팔로워 1000+ 보유한 한국인 크리에이터 30명 선발, 얼리액세스 제공 |

## 3. 각 항목별 핵심 콘텐츠 방향

### 3-1. 시리즈에 올라와야 하는 콘텐츠

| 카테고리 | 추천 콘텐츠 유형 | 이유 |
|----------|-----------------|------|
| 시네마틱 | AI 단편 영화 (3~5 에피소드) | 시리즈 구조의 존재 의미를 가장 잘 보여줌 |
| SF/판타지 | 세계관 기반 연작 | 팬덤 형성 가능성 높음 |
| 튜토리얼 | 모델별 사용법 시리즈 | 실용적 가치 → 반복 방문 유도 |
| 애니메이션 | AI 애니메이션 웹시리즈 | 일본/한국 서브컬처 팬층 유입 가능 |

**핵심 원칙**: 시리즈는 "이 크리에이터의 다음 편이 보고 싶다"는 기대감을 만들 수 있는 콘텐츠만.

### 3-2. 에피소드에 올라와야 하는 콘텐츠

| 카테고리 | 추천 콘텐츠 유형 | 이유 |
|----------|-----------------|------|
| 시네마틱 | 1~3분 내외 AI 영상 작품 | 완결성 있는 단위 영상 |
| 음악 | AI 뮤직비디오 | 음악 1곡 = 1에피소드로 자연스러운 매핑 |
| 로맨스/드라마 | 감성 숏필름 | 스토리 완결 단위 |
| 튜토리얼 | 특정 기법 딥다이브 | 한 가지 주제를 깊이 다루는 영상 |

**핵심 원칙**: 에피소드는 "하나의 완결된 이야기/주제"를 담는 단위. 시작과 끝이 있어야 함.

### 3-3. 클립에 올라와야 하는 콘텐츠

| 카테고리 | 추천 콘텐츠 유형 | 이유 |
|----------|-----------------|------|
| 코미디/밈 | AI 밈 영상 | 숏폼 + 유머 = 바이럴 최적 |
| 댄스 | AI 캐릭터 댄스 | 현재 AI 영상 트렌드와 정확히 맞음 |
| 동물 | AI 동물 영상 | 보편적 호감도, 낮은 진입장벽 |
| 게임 | AI로 만든 게임 시네마틱/트레일러 | 게이머 커뮤니티 유입 |
| SF/판타지 | 비주얼 쇼케이스 (스토리 없이 비주얼만) | "와" 하는 반응을 이끌어내는 임팩트 영상 |

**핵심 원칙**: 클립은 "스크롤하다가 멈추게 만드는" 콘텐츠. 5~15초, 강렬한 첫 프레임, 프롬프트 궁금증 유발.

### 3-4. 라운지에 올라와야 하는 콘텐츠

| 카테고리 | 추천 콘텐츠 유형 | 이유 |
|----------|-----------------|------|
| 화제의 AI 영상 | 유튜브/틱톡에서 화제가 된 AI 영상 링크 공유 | 라운지 방문 핵심 동기. "이거 봤어?" |
| AI 도구 쇼케이스 | 각 AI 도구(Sora, Runway 등)의 공식 데모/쇼릴 링크 | 새 도구/기능 발표 시 빠른 공유 |
| 크리에이터 발굴 | 유튜브/X에서 활동하는 AI 크리에이터의 작품 링크 | 외부 크리에이터를 Glip 생태계로 유도하는 퍼널 |
| Before/After 비교 | 다른 플랫폼에서 찾은 AI 영상 제작 과정 영상 | 학습 콘텐츠 수요 충족 |
| 트렌드 큐레이션 | "이번 주 AI 영상 트렌드" 형태의 외부 링크 모음 | 정기적 방문 습관 형성 |

**핵심 원칙**: 라운지는 "외부의 좋은 AI 영상을 발견하고 공유하는 공간". 직접 영상을 만들지 않아도 커뮤니티에 기여할 수 있는 가장 쉬운 경로. 초기에는 운영진이 적극적으로 양질의 외부 영상을 큐레이션하여 라운지의 품질 기준을 세워야 함.

## 4. 런칭 전 콘텐츠 시딩(Seeding) 전략

### 4-1. 최소 콘텐츠 물량 기준

| 항목 | 최소 수량 | 이유 |
|------|----------|------|
| 시리즈 | 10개 (각 3+ 에피소드) | 시리즈 탭이 비어 보이지 않는 최소 기준 |
| 에피소드 | 50개 | 시리즈 내 에피소드 + 독립 에피소드 포함 |
| 클립 | 200개 | 홈 피드 스크롤 시 콘텐츠가 끊기지 않는 기준 |
| 라운지 글 | 50개 | 외부 AI 영상 링크 큐레이션 최소 기준. 유튜브/틱톡 등에서 검증된 AI 영상을 선별하여 공유 |

### 4-2. 시딩 콘텐츠 제작 방법

#### Phase 1: 내부 제작 (런칭 4주 전)

**전담 시딩 팀 구성 (2~3명)**
- Sora, Kling, Runway, Pika, Luma 등 주요 모델로 직접 영상 제작
- 카테고리별 균형 배분 (시네마틱 30%, 밈/코미디 20%, SF 15%, 튜토리얼 15%, 기타 20%)

**구체적 제작 가이드**:

```
[클립 시딩 - 주력]
- 5~15초 영상 200개
- "임팩트 퍼스트" 원칙: 첫 1초에 시선을 잡는 영상
- 프롬프트/모델 정보 반드시 기입 (이것이 차별점)
- 카테고리별 배분:
  - 시네마틱 비주얼: 40개
  - AI 캐릭터/댄스: 30개
  - 밈/코미디: 30개
  - SF/판타지: 25개
  - 동물: 20개
  - 음악: 20개
  - 기타: 35개

[에피소드 시딩]
- 30초~3분 영상 50개
- 스토리 있는 단편 위주
- 튜토리얼 10개 필수 포함 (신규 유저 유입 경로)

[시리즈 시딩]
- 대표 시리즈 10개 기획
  예시:
  1. "서울 2099" (SF 시네마틱 시리즈, 5편)
  2. "AI로 만드는 뮤직비디오" (튜토리얼 시리즈, 4편)
  3. "Prompt Battle" (같은 프롬프트 다른 모델 비교, 4편)
  4. "60초 호러" (호러 숏필름 시리즈, 3편)
  5. "AI 동물원" (동물 영상 시리즈, 3편)
```

#### Phase 2: 크리에이터 초대 (런칭 2주 전)

  - 런칭 후 홈 추천 영역 우선 노출
  - 프리미엄 기능 6개월 무료 (향후 유료 기능 출시 시)
- 요청 사항: 최소 클립 5개 + 에피소드 1개 업로드

#### Phase 3: 운영 콘텐츠 (런칭 1주 전)

**라운지 시딩**:
- 운영진 계정으로 외부 AI 영상 링크 50개 이상 큐레이션
- 예시 콘텐츠:
  - 유튜브에서 화제가 된 AI 단편 영화 링크
  - 틱톡에서 바이럴된 AI 댄스/밈 영상 링크
  - Sora/Runway 공식 데모 영상 링크
  - 해외 AI 크리에이터의 작품 링크 (한줄 소개와 함께)
  - "이번 주 AI 영상 Top 5" 같은 큐레이션 포스트

**홈페이지 큐레이션**:

- 인기 라운지: 댓글 활성화된 스레드 우선 노출

### 4-3. 시딩 시 반드시 지켜야 할 원칙

1. **프롬프트 정보 100% 기입**: Glip의 차별점은 "프롬프트가 보이는 영상 플랫폼". 시딩 콘텐츠부터 이 원칙을 지켜야 유저가 문화를 학습함
2. **가짜 engagement 금지**: 좋아요/댓글 조작은 장기적으로 커뮤니티 신뢰 파괴. 차라리 물량이 적더라도 진짜 반응만
3. **다양한 모델 커버**: 특정 AI 모델에 편중되지 않도록. Sora, Kling, Runway, Pika, Luma, Hailuo 등 골고루
4. **퀄리티 하한선 설정**: "AI가 만든 어설픈 영상" 이미지가 붙으면 치명적. 시딩 단계에서는 상위 20% 퀄리티만 게시
5. **업로드 시점 분산**: 한 번에 200개 올리지 말고 2주에 걸쳐 매일 15~20개씩 분산 업로드 → "활발한 커뮤니티" 인상
