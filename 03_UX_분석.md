> 분석 일자: 2026-04-05
> 분석 관점: 시니어 UX 디자이너

## 1. 시리즈/에피소드/클립/라운지 컨셉 직관성 분석

### 1-1. 시리즈 (Series)

| 항목 | 평가 |
|------|------|
| 컨셉 전달력 | 중간 |
| 문제점 | "시리즈"라는 단어가 넷플릭스 등 OTT에서의 연속극을 연상시키지만, AI 생성 콘텐츠 맥락에서는 "같은 세계관/주제의 묶음"에 가까움. 이 차이를 설명하는 장치가 부족함 |

- 시리즈 페이지 자체는 존재하나, "시리즈가 무엇인지"에 대한 설명 텍스트나 빈 상태(empty state) 안내가 부재
- 업로드 페이지에서만 "1분 이상 영상을 에피소드로 올리고, 프로필에서 시리즈로 묶을 수 있다"는 설명이 등장 -- 탐색 페이지에서는 이 맥락이 전달되지 않음

### 1-2. 에피소드 (Episode)

| 항목 | 평가 |
|------|------|
| 컨셉 전달력 | 낮음 |
| 문제점 | 에피소드 페이지가 404 오류를 반환하고 있어 알파 단계에서 미구현 상태. 네비게이션에 노출된 메뉴가 작동하지 않으면 서비스 신뢰도에 직접적 타격 |

- "에피소드 = 1분 이상 개별 영상"이라는 정의는 업로드 플로우에서만 확인 가능
- 에피소드와 클립의 차이가 "길이(1분 기준)"뿐이라면, 사용자 입장에서 둘을 별도 메뉴로 분리한 이유가 직관적이지 않음

### 1-3. 클립 (Clip)

| 항목 | 평가 |
|------|------|
| 컨셉 전달력 | 높음 |
| 강점 | "1분 미만 짧은 영상"이라는 컨셉은 TikTok/Shorts/Reels 등의 학습 효과로 대부분의 사용자가 즉시 이해 가능 |

- 카테고리 필터(전체, 최신, 인기, 시네마틱, SF/판타지 등)가 잘 갖추어져 있음
- 클립 페이지가 가장 완성도 높은 탐색 경험을 제공

### 1-4. 라운지 (Lounge)

| 항목 | 평가 |
|------|------|
| 컨셉 전달력 | 중간 |
| 강점 | 외부 AI 영상(유튜브, 틱톡 등)의 임베드 링크 공유 공간이라는 컨셉은 전략적으로 우수. 직접 영상을 만들기 어렵지만 커뮤니티에 기여하고 싶은 유저를 위한 진입점 역할 |

- 라운지의 핵심 목적: (1) 직접 제작 역량 없는 유저의 참여 허들 낮추기, (2) 서비스 초기에 이미 검증된 외부 영상으로 플랫폼 콘텐츠 물량 확보
- 다만 "라운지"라는 네이밍만으로는 "외부 영상 공유 공간"이라는 점이 즉시 전달되지 않을 수 있음
- 홈에서 "인기 라운지"로 노출될 때, 내부 제작 콘텐츠(시리즈/에피소드/클립)와의 차별점이 시각적으로 구분되면 좋겠음 (예: 외부 플랫폼 아이콘 표시)

### 개선 제안

1. **각 섹션 랜딩에 한 줄 설명 배치**: "에피소드는 1분 이상의 AI 생성 영상입니다" 같은 서브헤딩으로 컨셉 즉시 전달
2. **라운지 서브 설명 추가**: "라운지" 네이밍 자체는 유지하되, 섹션 진입 시 "유튜브, 틱톡 등 외부 AI 영상을 발견하고 공유하는 공간" 같은 한 줄 설명 배치. 또는 "발견" 같은 네이밍도 검토 가능
3. **미구현 페이지 비활성화**: 404가 뜨는 에피소드 페이지는 네비게이션에서 임시 제거하거나 "준비 중" 표시

## 2. 정보 구조(IA) 적절성 분석

### 2-1. 현재 IA 구조

```
홈 (피드)
├── 주간 인기 시리즈
├── 주간 인기 에피소드
├── 주간 인기 클립
├── 추천 시리즈 / 에피소드 / 클립
└── 인기 라운지

시리즈 ─── 에피소드(1분+) ─── 클립(1분-)
                                    ↑ 시리즈에 포함 불가

라운지 (외부 링크 공유 + 커뮤니티)
팔로잉 (구독 피드)
업로드
프로필
```

### 2-2. 계층 구조 문제점

**문제 1: 3단 계층이 과도함**
- 시리즈 > 에피소드 > 클립은 TV 프로덕션 구조를 차용했으나, AI 생성 콘텐츠의 특성과 맞지 않음
- AI 생성 영상은 대부분 짧고, 독립적이며, 시리즈로 기획하는 경우가 드묾
- YouTube도 채널 > 영상의 2단 구조이며, 재생목록(플레이리스트)은 선택적 기능

**문제 2: 에피소드와 클립 구분의 모호성**
- 1분이라는 기준이 사용자의 멘탈 모델과 일치하지 않음
- 59초 영상은 클립, 61초 영상은 에피소드 -- 이 구분이 탐색 경험에 실질적 가치를 제공하는지 의문
- 사용자는 "짧은 영상 / 긴 영상"이 아니라 "재미있는 영상 / 내 관심 장르"로 탐색함

**문제 3: 네비게이션 과부하**
- 8개 메뉴 항목(홈, 시리즈, 에피소드, 클립, 라운지, 팔로잉, 업로드, 프로필)은 사이드바에서 인지 부하를 유발
- 시리즈/에피소드/클립이 모두 독립 메뉴로 존재할 필요가 있는지 재검토 필요

### 2-3. 개선 제안

**안 A: 2단 구조로 단순화**
```
탐색 (홈)
├── 영상 (길이 무관, 필터로 분류)
├── 시리즈 (묶음 콘텐츠)
└── 라운지 (커뮤니티)
```
- "에피소드"와 "클립"을 통합하여 "영상"으로 일원화
- 시리즈는 영상을 묶는 플레이리스트 개념으로 유지
- 길이별 정렬은 필터/탭으로 제공

**안 B: 유지하되 탭으로 전환**
```
탐색 (홈)
├── 콘텐츠 [시리즈 | 에피소드 | 클립] ← 탭 전환
├── 라운지
└── 팔로잉
```
- 3개 콘텐츠 유형을 하나의 메뉴 안에서 탭으로 전환
- 네비게이션 항목 수를 5개 이하로 축소

## 3. 신규 유저 온보딩 경험 분석

### 3-1. 현재 상태

| 온보딩 요소 | 존재 여부 |
|-------------|-----------|
| 환영 메시지 / 투어 | 없음 |
| 서비스 소개 화면 | 없음 |
| 관심 카테고리 선택 | 없음 |
| 샘플 콘텐츠 추천 | 부분적 (주간 인기/추천 섹션) |
| 업로드 가이드 | 업로드 페이지 내 간략 설명만 존재 |
| 용어 설명 | 없음 |

### 3-2. 핵심 문제

**Cold Start 문제**
- 신규 유저가 홈에 도착하면 "주간 인기 시리즈/에피소드/클립"이 나열되지만, 이 세 가지가 무엇이 다른지 설명이 없음
- AI 생성 콘텐츠 커뮤니티라는 서비스 특성상, 일반 영상 플랫폼과 차별화된 가치(프롬프트 공유, 모델 정보 등)를 초기에 전달해야 하나 이 부분이 누락

**로그인 벽의 부재 또는 존재 모호성**
- 로그인 없이 탐색이 가능한지, 어떤 기능에서 로그인이 요구되는지 명확하지 않음
- 업로드, 팔로잉, 프로필은 당연히 로그인이 필요하겠지만, 이 전환 시점의 경험이 매끄러운지 확인 필요

### 3-3. 개선 제안

1. **첫 방문 시 3초 인트로 오버레이**
   - "Glip은 AI 크리에이터를 위한 영상 공유 커뮤니티입니다"
   - 시리즈/에피소드/클립/라운지를 아이콘+한 줄 설명으로 소개
   - "탐색 시작하기" CTA 버튼

2. **관심 카테고리 선택 (선택적)**
   - 시네마틱, SF/판타지, 애니메이션 등 13개 카테고리 중 3개 이상 선택
   - 홈 피드 개인화에 활용

3. **첫 업로드 유도 플로우**
   - 프로필 완성 후 "첫 클립을 올려보세요" 넛지
   - 업로드 페이지에 단계별 가이드 (파일 선택 > 정보 입력 > AI 분석 > 게시)

4. **프롬프트/모델 정보의 가치 시각화**
   - 홈에서 "이 영상은 [Sora v2] + [프롬프트 미리보기]로 만들어졌습니다" 같은 카드를 노출
   - 서비스의 핵심 차별점을 바로 보여주는 것이 중요

## 4. 콘텐츠 탐색/발견 UX 개선점

### 4-1. 현재 탐색 구조

```
검색 (상단 검색바)
카테고리 필터 (클립 페이지에서 확인: 전체/최신/인기/장르별)
큐레이션 섹션 (홈: 주간 인기, 추천)
팔로잉 피드
```

### 4-2. 문제점

**문제 1: 탐색 진입점 분산**
- 시리즈/에피소드/클립이 각각 별도 페이지로 존재하여, 사용자가 "좋은 AI 영상을 보고 싶다"는 단순한 욕구를 충족하려면 3개 페이지를 오가야 함
- 홈이 이를 통합하려 하지만, 각 섹션이 수평 스크롤 카드 리스트라면 깊이 있는 탐색이 어려움

**문제 2: 카테고리의 불균형**
- 13개 + 성인(19+) = 14개 카테고리는 관리와 탐색 양쪽에서 부담
- "시네마틱"과 "로맨스/드라마"의 경계가 모호하고, "튜토리얼"은 성격이 완전히 다른 콘텐츠
- AI 생성 콘텐츠 특성상 "사용 모델별" 또는 "생성 도구별" 필터가 더 유용할 수 있음

**문제 3: 검색의 한계**
- 검색바가 존재하나, AI 콘텐츠 플랫폼에서는 "프롬프트 검색"이나 "모델명 검색"이 핵심 기능이 되어야 함
- 일반 키워드 검색만으로는 AI 크리에이터 커뮤니티의 차별화된 탐색 경험을 제공하기 어려움

**문제 4: 추천 알고리즘의 투명성 부재**
- "추천 시리즈/에피소드/클립"이 어떤 기준으로 추천되는지 사용자에게 전달되지 않음
- "당신이 좋아할 SF 클립" vs. 단순 "추천 클립" -- 전자가 클릭률과 신뢰도 모두 높임

### 4-3. 개선 제안

1. **통합 피드 + 필터 패턴 도입**
   - 홈에서 모든 콘텐츠 유형을 하나의 피드로 보여주되, 상단 필터 칩으로 "시리즈만 / 에피소드만 / 클립만 / 라운지" 전환
   - Pinterest/Instagram Explore 스타일의 무한 스크롤 그리드

2. **AI 특화 필터 추가**
   - 사용 모델 (Sora, Runway, Kling, Pika 등)
   - 생성 방식 (텍스트→영상, 이미지→영상, 모션 트랜스퍼 등)
   - 프롬프트 포함 여부 ("프롬프트 공개" 필터)

3. **카테고리 구조 개편**
   - 현재 14개를 8개 이내로 축소
   - "장르" (SF, 호러, 로맨스 등)와 "형식" (시네마틱, 애니메이션, 튜토리얼 등)을 분리하여 2축 필터로 제공

4. **트렌딩 프롬프트/모델 섹션**
   - "이번 주 인기 프롬프트 Top 10"
   - "Sora로 만든 인기 클립" 같은 모델 기반 큐레이션
   - AI 크리에이터 커뮤니티의 핵심 가치를 탐색 경험에 반영

## 5. 핵심 사용자 플로우별 문제점과 개선 제안

### 5-1. 콘텐츠 소비 플로우 (시청자)

```
홈 진입 → 콘텐츠 발견 → 시청 → 반응(좋아요/댓글) → 추가 탐색
```

| 단계 | 문제점 | 심각도 | 개선 제안 |
|------|--------|--------|-----------|
| 홈 진입 | 6개 섹션이 동시 로딩되어 시각적 과부하 | 중 | 핵심 1-2개 섹션을 히어로로 강조, 나머지는 스크롤 하단에 배치 |
| 콘텐츠 발견 | 시리즈/에피소드/클립 구분이 탐색을 방해 | 높 | 통합 피드 + 유형 필터 |
| 시청 | 시청 경험 미확인 (알파 한계) | - | 인라인 재생, 프롬프트 정보 사이드패널 |
| 추가 탐색 | 관련 콘텐츠 추천 미확인 | - | "이 프롬프트와 유사한 클립" 추천 |

### 5-2. 콘텐츠 업로드 플로우 (크리에이터)

```
업로드 클릭 → 유형 선택 (에피소드/클립/라운지) → 파일 업로드 → 정보 입력 → AI 분석 → 게시
```

| 단계 | 문제점 | 심각도 | 개선 제안 |
|------|--------|--------|-----------|
| 유형 선택 | 에피소드 vs 클립을 사용자가 직접 선택해야 함. 1분 기준을 외울 필요 | 중 | 파일 업로드 후 자동 분류, 또는 통합 업로드 후 시스템이 판단 |
| 파일 업로드 | MP4/MOV/WebM, 500MB 제한은 적절 | 낮 | 드래그앤드롭 지원 확인됨 (양호) |
| AI 분석 | "AI 분석 후 공개"라는 안내가 있으나 소요 시간, 분석 내용이 불명확 | 높 | 예상 소요 시간 표시, 분석 진행률 표시, 분석 완료 알림 |
| 라운지 업로드 | 외부 링크 공유인데 "업로드"라는 워딩과 불일치 | 중 | "링크 공유" 또는 "글쓰기"로 분리 |

### 5-3. 시리즈 구성 플로우 (크리에이터)

```
에피소드 여러 개 업로드 → 프로필에서 시리즈로 묶기
```

| 단계 | 문제점 | 심각도 | 개선 제안 |
|------|--------|--------|-----------|
| 전체 플로우 | 시리즈 생성이 업로드와 분리되어 있어 2단계 작업이 필요 | 높 | 업로드 시 "기존 시리즈에 추가" 옵션 제공, 또는 업로드 완료 후 즉시 시리즈 연결 제안 |
| 발견성 | 시리즈 구성 기능이 프로필 내에 숨겨져 있어 인지하기 어려움 | 높 | 업로드 플로우 내에서 시리즈 연결을 자연스럽게 유도 |

### 5-4. 외부 영상 공유 플로우 (라운지)

```
라운지 진입 → 외부 AI 영상 탐색 → 임베드 영상 시청 → 댓글/반응 → 자신도 링크 공유
```

| 단계 | 문제점 | 심각도 | 개선 제안 |
|------|--------|--------|-----------|
| 진입 | "라운지"라는 용어만으로 외부 영상 공유 공간임을 인지하기 어려움 | 중 | 서브헤딩으로 "유튜브, 틱톡 등 외부 AI 영상을 발견하고 공유하세요" 안내 |
| 링크 공유 | 외부 링크 공유 UX가 직관적인지 확인 필요. "업로드"라는 워딩보다 "링크 공유" 또는 "영상 발견" 워딩이 적합 | 중 | 업로드와 라운지 공유를 명확히 분리. 라운지는 "공유하기" 버튼으로 |
| 참여 동기 | 직접 제작을 안 해도 기여할 수 있다는 점이 라운지의 핵심 장점 | - | 이 장점을 강조: "영상 제작은 못해도 좋은 영상을 알아보는 안목이 있다면!" |
| 전환 유도 | 라운지에서 외부 영상만 보는 유저를 내부 크리에이터로 전환하는 경로 부재 | 중 | "이런 영상을 직접 만들어보세요" → 클립 업로드 유도 CTA 배치 |

### 5-5. 팔로잉/구독 플로우

```
크리에이터 프로필 방문 → 팔로우 → 팔로잉 피드에서 업데이트 확인
```

| 단계 | 문제점 | 심각도 | 개선 제안 |
|------|--------|--------|-----------|
| 팔로잉 피드 | 독립 메뉴로 존재하나, 팔로우한 크리에이터가 없으면 빈 화면 예상 | 중 | 빈 상태에서 "인기 크리에이터 추천" 제공 |
| 발견 경로 | 크리에이터를 발견하는 경로가 콘텐츠 → 프로필 경유뿐 | 중 | "인기 크리에이터" 또는 "이달의 크리에이터" 섹션 홈에 추가 |

## 종합 평가 및 우선순위 로드맵

### 심각도별 정리

**즉시 수정 (P0)**
- 에피소드 페이지 404 오류 해결 (또는 메뉴 임시 제거)
- 각 콘텐츠 유형의 한 줄 설명 추가

**단기 개선 (P1)**
- 에피소드/클립 구분 재검토 및 통합 피드 도입
- 업로드 시 시리즈 연결 플로우 개선
- AI 분석 대기 시간 투명성 확보
- 신규 유저 온보딩 인트로 추가

**중기 개선 (P2)**
- AI 특화 탐색 기능 (모델별, 프롬프트 검색)
- 카테고리 구조 개편 (14개 → 8개 이내)
- 라운지→크리에이터 전환 유도 기능 (외부 영상 소비자를 내부 제작자로)
- 크리에이터 발견 경로 다양화

**장기 비전 (P3)**
- 개인화 추천 알고리즘 고도화
- 프롬프트 마켓플레이스 가능성 탐색
- 크리에이터 대시보드 (분석/인사이트)

### 핵심 인사이트

> Glip의 가장 큰 UX 과제는 "AI 생성 콘텐츠 커뮤니티"라는 독특한 포지셔닝의 가치가 현재 인터페이스에 충분히 반영되지 않았다는 점이다. 시리즈>에피소드>클립이라는 TV 프로덕션 차용 구조보다, **프롬프트와 모델 정보를 중심으로 한 탐색 구조**가 이 서비스의 본질에 더 부합한다. 콘텐츠의 "길이"가 아니라 "어떻게 만들었는가"가 이 커뮤니티의 핵심 관심사이기 때문이다.
>
> 한편, **라운지(외부 영상 링크 공유)는 전략적으로 매우 우수한 설계**다. 직접 제작 → 간접 기여(라운지) → 소비의 3단계 참여 구조를 통해 다양한 유저층을 포용하며, 초기 콘텐츠 물량 확보에도 효과적이다. 다만 이 구조의 가치가 UI에서 명확히 전달되도록 라운지의 목적과 역할을 시각적으로 차별화할 필요가 있다.
