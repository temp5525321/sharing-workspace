# 에피소드 / 클립 구조 재설계 검토

> 작성일: 2026-04-06
> 배경: 시리즈는 v2로 이관 결정. 에피소드·클립 두 축의 구분 기준 재정의 필요.

---

## 현재 구조의 문제점

| 항목 | 현재 규칙 | 문제 |
|------|----------|------|
| 에피소드 | 1분 이상, 가로/세로 모두 가능 | 59초 vs 61초 구분에 실질적 의미 없음 |
| 클립 | 1분 이하, 세로만 가능 | AI 도구 대부분 가로(16:9) 출력 → 업로드 허들 |
| 분류 방식 | 시간 기반 자동 분류 | 크리에이터 의도와 무관한 기계적 분류 |

---

## 4개 관점 분석

### 1. 기획자 관점

**"1분" 기준은 폐기를 권장합니다.**

AI 영상 엔진들의 실제 출력 길이:
- Sora: 최대 60초 / Runway: 10초 단위 / Kling: 최대 120초 / Pika: 짧은 클립 위주

1분이라는 기준은 **플랫폼이 정한 임의적 숫자**이지, 크리에이터나 소비자의 멘탈 모델에 기반한 구분이 아닙니다.

**구분 기준 제안: 스토리 연결성**

| 구분 | 에피소드 | 클립 |
|------|---------|------|
| 핵심 속성 | 이어지는 이야기의 한 "화" | 독립적인 한 편의 영상 |
| 길이 | 제한 없음 | 제한 없음 |
| 비율 | 제한 없음 | 제한 없음 |
| 관계 | 다른 에피소드와 연결됨 (1화→2화) | 단독으로 완결 |

에피소드 = "다음 화가 있는 콘텐츠", 클립 = "그 자체로 완결되는 콘텐츠"

이렇게 하면 08_콘텐츠_피라미드에서 설계한 플라이휠(클립으로 유입 → 에피소드로 체류)이 자연스럽게 작동합니다.

**리스크**: 시리즈를 빼면서 에피소드에 "연결" 기능을 부여하면, 사실상 에피소드가 미니 시리즈 역할을 하게 됨. v1에서 오히려 복잡도를 높일 수 있음.

**대안**: v1에서는 클립만 주력으로 가고, 에피소드는 "클립 묶음(플레이리스트)" 개념으로 단순화하는 것도 고려 가능.

---

### 2. UX 디자이너 관점

**세로 제한은 즉시 해제해야 합니다.**

현실적인 이유:
- AI 영상 도구들은 대부분 **16:9 (가로)**를 기본 출력
- 크리에이터가 세로 변환 작업을 추가로 해야 한다면 업로드를 포기할 가능성 높음
- 틱톡/릴스는 세로가 표준이지만, Glip은 "AI 크리에이터 커뮤니티"이지 "숏폼 플랫폼"이 아님

**UX 관점의 구조 제안:**

```
현재:  에피소드(1분+, 가로/세로) | 클립(1분-, 세로만)  ← 규칙이 복잡
제안:  영상(길이/비율 자유) + 탐색 시 필터로 구분       ← 업로드는 하나, 소비는 필터
```

업로드 시 크리에이터에게 물어볼 것은 딱 하나:
**"이 영상은 이전 영상의 다음 화인가요?"** (예/아니오)
- "예" → 에피소드로 연결
- "아니오" → 독립 클립

길이와 비율은 시스템이 자동 감지. 유저가 분류를 고민하게 만들지 않아야 함.

---

### 3. 마케터 관점

**바이럴 관점에서 "제한"은 항상 손해입니다.**

문제 시나리오:
1. Sora로 60초짜리 가로 영상을 만들었는데, Glip에서는 "에피소드"로 분류됨 → 크리에이터: "나는 그냥 클립 하나 올리고 싶었는데?"
2. 클립이 세로만 가능하면, 가로로 만든 AI 영상은 클립으로 못 올림 → 크리에이터의 콘텐츠가 플랫폼 규칙에 의해 거부당하는 경험

마케팅에서 가장 중요한 건 **"첫 업로드까지의 마찰을 최소화"**하는 것.

| 항목 | 현재 | 제안 |
|------|------|------|
| 분류 기준 | 시간(1분) + 비율(세로) | 스토리 연결 여부만 |
| 업로드 플로우 | 유형 선택 → 제약 확인 → 실패 가능 | 그냥 올리기 → 시스템이 알아서 |
| 바이럴 효과 | 제한 때문에 업로드 포기 발생 | 모든 AI 영상이 올라올 수 있음 |

**소비 경험에서는 숏폼 느낌을 살릴 수 있음**: 세로 영상은 자동으로 풀스크린 스와이프 UI, 가로 영상은 일반 플레이어. 이건 업로드 제한이 아니라 재생 UX의 문제.

---

### 4. AI 영상 제작자 (감독) 관점

**현장의 목소리: "1분 제한도, 세로 제한도 현실과 안 맞습니다"**

현재 AI 영상 제작의 현실:
- 대부분의 결과물은 5~30초 (도구 한 번 실행 = 1 클립)
- 가로(16:9)가 기본 출력. 시네마틱은 2.39:1까지 사용
- 에피소드를 만들려면 여러 클립을 편집 소프트웨어에서 이어붙여야 함 → 크리에이터의 후가공 영역

AI 영상 제작자 입장에서:
- "클립" = AI 도구에서 나온 원본 한 편 (5~30초, 대부분 가로)
- "에피소드" = 클립 여러 개를 편집한 작품 (30초~수분, 비율 자유)

**1분 기준의 문제**: Kling으로 120초짜리 댄스 영상을 뽑으면 클립인가 에피소드인가? 제작자 입장에서는 "그냥 AI가 뽑은 한 편"이니 클립인데, 시스템은 에피소드로 분류.

**세로 제한의 문제**: 시네마틱/SF/호러 등 Glip의 핵심 장르는 거의 다 가로. 세로는 댄스/밈 정도. 클립을 세로로 제한하면 가장 임팩트 있는 콘텐츠가 클립 피드에서 빠짐.

## 종합 권장안

4개 관점이 **모두 동일한 방향**을 가리키고 있음:

### 에피소드 vs 클립 새 구분 기준

| | 현재 | 권장 |
|--|------|------|
| 구분 기준 | 길이(1분) | **스토리 연결 여부** |
| 클립 비율 | 세로만 | **제한 없음** |
| 에피소드 비율 | 가로/세로 | **제한 없음** |
| 길이 제한 | 있음 | **없음** (업로드 용량만 제한) |

### v1 콘텐츠 구조

- **클립** = 독립적인 AI 영상 한 편. 길이/비율 무관. Glip의 주력 콘텐츠.
- **에피소드** = 이어지는 이야기. "1화 → 2화 → 3화" 구조. 클립들을 연결하는 상위 개념.
- **시리즈** = v2에서 도입 (합의 완료)
- **라운지** = 현행 유지 (외부 링크)
