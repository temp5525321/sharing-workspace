## 3. 기술적 품질 체크

## 5. Critical 이슈 (런칭 전 반드시 수정)

### C-01. 주요 메뉴 페이지 404 오류
- **심각도**: Critical
- **대상**: /series, /episodes, /clips, /lounge, /following
- **설명**: 메인 네비게이션에 노출되는 5개 핵심 메뉴가 모두 404를 반환함. 사용자가 첫 방문 시 내비게이션 클릭만으로 즉시 깨진 페이지를 만나게 됨.
- **참고**: Playwright로 실제 브라우저 캡처 결과, /series와 /lounge는 정상 작동 확인됨. /clip(클립)은 실제로도 404. WebFetch와 실제 브라우저 결과가 다른 것은 Next.js CSR/SSR 차이로 추정.
- **실제 404 확인 페이지**: /clip (클립 목록 페이지)
- **조치**: /clip 라우트 우선 수정 필요. 나머지는 실제 브라우저에서 재확인 후 판단

### C-02. og:image / twitter:image 미설정
- **심각도**: Critical
- **설명**: SNS 공유 시 이미지가 표시되지 않음. AI 영상/이미지 커뮤니티라는 서비스 특성상 시각적 미리보기가 매우 중요
- **조치**: 대표 OG 이미지 제작 및 메타태그 추가 (권장 1200x630px)

### C-03. 로그인 페이지(/login) 404
- **심각도**: Critical
- **설명**: /login URL로 직접 접근 불가. 딥링크 또는 이메일 인증 리다이렉트 시 문제 발생 가능
- **조치**: /login 라우트 구현 또는 홈으로 리다이렉트 처리

### C-04. 프로필 페이지(/profile) 404
- **심각도**: Critical
- **설명**: 로그인 상태에서도 /profile 접근 불가 추정. 비로그인 시 로그인 유도 페이지 필요
- **조치**: 인증 상태에 따른 분기 처리 구현

### C-05. sitemap에 404 페이지 포함 가능성
- **심각도**: Critical (SEO)
- **설명**: robots.txt에서 sitemap.xml 참조하고 있으나, sitemap 내 URL이 실제로 유효한지 확인 필요. 404 페이지가 포함되어 있으면 검색엔진 크롤링 품질 저하
- **조치**: sitemap 생성 로직에서 실제 존재하는 페이지만 포함하도록 수정

## 6. 권장 수정 사항 (Nice-to-have)

### N-01. apple-touch-icon 추가
- **설명**: iOS 홈 화면 추가 시 아이콘 미표시
- **조치**: 180x180px apple-touch-icon 추가

### N-02. theme-color 메타태그 추가
- **설명**: 모바일 브라우저 주소창 색상 미설정
- **조치**: `<meta name="theme-color" content="#...">` 추가 (다크모드 대응 포함)

### N-03. PWA manifest 추가
- **설명**: manifest.json/site.webmanifest 미설정
- **조치**: 오프라인 경험 및 앱 설치 유도를 위해 manifest 추가 검토

### N-04. 저작권 연도 자동화
- **설명**: 푸터의 "© 2024 Glip"이 현재 연도(2026)와 불일치
- **조치**: `new Date().getFullYear()` 등으로 동적 처리 또는 "© 2024-2026 Glip"으로 수정

### N-05. JSON-LD 구조화 데이터 추가
- **설명**: 검색엔진 리치 스니펫 미지원
- **조치**: Organization, WebSite, VideoObject 등 스키마 추가

### N-06. 검색 입력 필드 접근성 개선
- **설명**: 검색 input에 label 또는 ARIA 속성 누락
- **조치**: `aria-label="검색"` 또는 시각적으로 숨긴 `<label>` 추가

### N-07. robots meta 태그 추가
- **설명**: HTML head에 robots meta 미설정 (robots.txt만 존재)
- **조치**: `<meta name="robots" content="index, follow">` 추가

### N-08. /about, /notice 페이지 콘텐츠 작성
- **설명**: "준비 중" 상태로 노출되어 미완성 인상
- **조치**: 최소한의 서비스 소개 및 공지사항 콘텐츠 작성 또는 런칭 전까지 네비게이션에서 숨김

### N-09. 쿠키 동의 배너 검토
- **설명**: 개인정보 관련 쿠키 동의 배너 미표시
- **조치**: 쿠키 정책 페이지는 존재하므로 (terms 내 링크) 실제 동의 UI 추가 검토

### N-10. 404 에러 페이지 커스터마이징
- **설명**: 기본 Next.js 404 메시지("This page could not be found") 영문 표시
- **조치**: 한국어 커스텀 404 페이지 구현 ("페이지를 찾을 수 없습니다" + 홈 이동 버튼)

### N-11. 에러 메시지 영문 노출
- **설명**: 한국어 서비스이나 404 에러 메시지가 영문으로 표시됨
- **조치**: not-found.tsx 커스텀 페이지에서 한국어 메시지 제공
