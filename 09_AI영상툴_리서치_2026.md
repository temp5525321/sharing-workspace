## 7. GitHub 협업 세팅 (두 명의 Claude Code 유저용)

### 방법 1: 공유 저장소 (추천)
```bash
# 1. GitHub에 저장소 생성
gh repo create glip-research --private --clone

# 2. 마크다운 파일 추가
cp /Users/daddy/Dev/Glip/*.md ./
git add . && git commit -m "Initial research files"
git push origin main

# 3. 협업자 초대
gh api repos/{owner}/glip-research/collaborators/{username} -X PUT

# 4. 각자 작업 후 push/pull
git pull origin main  # 시작 전
# ... 작업 ...
git add . && git commit -m "Update research"
git push origin main
```

### 방법 2: Fork + PR 방식
- 한 명이 원본 저장소 소유
- 다른 한 명이 Fork -> 수정 -> Pull Request
- 충돌 방지에 더 안전

### 방법 3: Branch 분리
```bash
# 각자 브랜치에서 작업
git checkout -b user1-research
# ... 작업 ...
git push origin user1-research
# PR로 main에 병합
```

### Claude Code 팁
- 각 봇이 다른 파일을 담당하면 충돌 최소화
- `.gitignore`에 `node_modules/`, `.env` 등 추가
- 작업 전 항상 `git pull` 습관화
